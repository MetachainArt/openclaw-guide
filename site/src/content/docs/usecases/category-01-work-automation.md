<h2 id="case-1">1. 맞춤형 아침 브리핑 자동화</h2>

### 📖 상세 설명

매일 아침 정해진 시간(예: 6시 30분)에 이메일 요약, 오늘 일정, 관심 뉴스, 날씨, 어제 하던 작업의 맥락을 메신저로 자동 전송합니다. 하루를 시작하기 전 중요한 정보를 한눈에 확인할 수 있습니다.

**어떤 정보가 포함되나요?**
- 🌤️ 오늘 날씨 (기온, 강수확률, 미세먼지)
- 📧 미확인 이메일 요약 (중요도별 분류)
- 📅 오늘의 일정 (회의, 마감, 약속)
- 📰 주요 뉴스 (설정한 키워드 기반)
- 📊 어제의 업무 현황 (완료/진행중)
- 💡 오늘 해야 할 일 추천

### 🎯 활용 효과

- **매일 15-20분 절약**: 정보 수집 및 정리 시간 감소
- **중요한 이메일 놓침 방지**: AI가 중요도 판단하여 알림
- **하루 계획 수립 시간 단축**: 모든 정보가 한 메시지에

### 🛠️ 필요 스킬

| 스킬 | 용도 | 설치 명령어 |
|------|------|------------|
| email | 이메일 요약 | `clawhub install email` |
| calendar | 일정 확인 | `clawhub install calendar` |
| weather | 날씨 정보 | `clawhub install weather` |
| news | 뉴스 수집 | `clawhub install news` |

⚠️ **스킬 설치 전 [보안 체크리스트](/security/skill-checklist) 확인 필수!**

<h2 id="case-2">2. 이메일 인박스 제로 (Inbox Zero)</h2>

### 📖 상세 설명

수많은 이메일을 자동으로 분석하여 중요한 메일만 골라 요약하고, 뉴스레터는 자동으로 아카이브하며, 스팸은 정리합니다. 매일 수십 개씩 쌓이는 이메일에서 핵심만 추출하여 인박스를 깔끔하게 유지합니다.

**어떻게 작동하나요?**
1. AI가 이메일 내용을 분석
2. 중요도 판단 (긴급/중요/일반/뉴스레터/스팸)
3. 자동 분류 및 정리
4. 중요 메일만 요약하여 알림

### 🎯 활용 효과

- **인박스 0개 유지**: 자동 정리로 깔끔한 메일함
- **중요 메일 놓침 방지**: AI가 중요도 판단
- **메일 확인 시간 70% 단축**: 불필요한 메일 필터링

### 🛠️ 필요 스킬

| 스킬 | 용도 | 설치 명령어 |
|------|------|------------|
| gmail | Gmail 연동 | `clawhub install gmail` |
| filter | 메일 필터링 | `clawhub install filter` |
| classifier | 메일 분류 AI | `clawhub install classifier` |

**Step 3: 자동화 스크립트**

```javascript
// ~/.openclaw/skills/inbox-zero/processor.js
module.exports = async (agent) => {
  const emails = await agent.gmail.getUnread();
  const results = {
    processed: 0,
    important: [],
    archived: [],
    deleted: [],
    labeled: []
  };
  
  for (const email of emails) {
    // AI 분류
    const category = await agent.classifier.categorize(email, [
      'critical',
      'important',
      'normal',
      'newsletter',
      'spam'
    ]);
    
    switch(category) {
      case 'critical':
        await agent.gmail.flag(email.id, 'urgent');
        await agent.notify(`🚨 긴급 메일: ${email.subject}`, {
          priority: 'high'
        });
        results.important.push(email);
        break;
        
      case 'important':
        await agent.gmail.flag(email.id);
        results.important.push(email);
        break;
        
      case 'newsletter':
        await agent.gmail.archive(email.id);
        await agent.gmail.label(email.id, 'Newsletters');
        results.archived.push(email);
        break;
        
      case 'spam':
        await agent.gmail.trash(email.id);
        results.deleted.push(email);
        break;
        
      default:
        // normal - 그대로 유지
        break;
    }
    
    results.processed++;
  }
  
  // 요약 리포트
  if (results.processed > 0) {
    await agent.notify({
      title: '📧 인박스 정리 완료',
      message: `
처리: ${results.processed}개
🔴 긴급: ${results.important.filter(e => e.priority === 'urgent').length}개
⭐ 중요: ${results.important.filter(e => e.priority !== 'urgent').length}개
📦 아카이브: ${results.archived.length}개
🗑️ 삭제: ${results.deleted.length}개
      `,
      channel: 'telegram'
    });
  }
  
  return results;
};
```

<h2 id="case-3">3. 자동 이메일 초안 작성</h2>

### 📖 상세 설명

내 평소 말투와 스타일을 학습하여 중요 메일에 대한 답장 초안을 자동으로 작성합니다. 사용자의 과거 이메일 패턴을 분석해 자연스러운 톤으로 초안을 만들어 제시합니다.

### 🎯 활용 효과

- 답장 작성 시간 50% 단축
- 일관된 커뮤니케이션 톤 유지
- 번거로운 정중한 표현 자동 생성

### 🛠️ 필요 스킬

| 스킬 | 용도 | 설치 명령어 |
|------|------|------------|
| gmail | Gmail 연동 | `clawhub install gmail` |
| memory | 사용자 스타일 학습 | `clawhub install memory` |
| composer | 메일 작성 AI | `clawhub install composer` |

### 📝 단계별 설정

**Step 1: 학습 데이터 제공**

```bash
# 과거 메일 100개로 학습
openclaw memory train --source gmail --limit 100 --style communication
```

**Step 2: 초안 생성 규칙**

```json
{
  "composer": {
    "email": {
      "tone": "professional_but_friendly",
      "formality": "business_casual",
      "maxLength": 300,
      "signature": "\n\n감사합니다.\n${user.name}",
      "templates": {
        "meeting_request": "meeting_template",
        "follow_up": "followup_template",
        "thank_you": "thanks_template"
      }
    }
  }
}
```

### 💬 사용 예시

```
User: 이 메일 답장 초안 써줘
(첨부: "회의 일정 변경" 메일)

OpenClaw:
📧 답장 초안 작성 완료

---
제목: RE: 회의 일정 변경

안녕하세요 김대리님,

메일 확인했습니다. 
수요일 오후 2시로 변경 확인했으며, 그 시간에 맞춰
회의실 B로 참석하겠습니다.

혹시 회의 안건 미리 공유 가능하시면 
검토하고 준비해 가겠습니다.

감사합니다.
홍길동
---

🎨 말투 분석:
• 평소 비슷한 상황에서 사용한 표현 반영
• 정중하면서도 친근한 톤
• 추가 액션 제안 포함

[✅ 승인 후 전송] [✏️ 직접 수정] [🔄 다시 작성]
```

### 📝 단계별 설정

**Step 1: Google Calendar 연동**

```bash
clawhub install calendar
openclaw config calendar
```

```json
{
  "calendar": {

    "reminderMinutes": [10, 60],
    "timezone": "Asia/Seoul",
    "conflictDetection": true,
    "autoInvite": true
  }
}
```

```
User: 내일 오후 2시에 팀 미팅 잡아줘
```

<h2 id="case-9">9. 메일 운영 자동화 구축 가이드 (실전 구축 순서)</h2>

### 📖 상세 설명

여러 메일 계정을 하나로 모아, OpenClaw로 분류하고 요약하여 옵시디언(기록)과 Todoist(실행)까지 연결하는 풀 자동화 파이프라인 구축 가이드입니다. 

**목표**:
1. 여러 메일 계정(구글/네이버 등)을 수집 Gmail로 통합
2. Gmail 필터로 AI/Action, AI/Partnership, AI/Updates, AI/Archive, AI/Inbox 분류
3. OpenClaw가 메일을 읽어 옵시디언 리포트 + Todo 후보 생성
4. 고우선 액션(P1/P2)은 Todoist(휴대폰 앱)로 자동 전송

### 📝 단계별 설정 (실전 구축 순서)

**1) 메일 수집 구조 먼저 만들기**
- 수집 메일함 준비: 예) `gahee9308@gmail.com`를 메일 허브로 사용
- 기존 계정에서 전달 설정: 회사메일 제외한 계정들을 수집 메일로 전달되도록 자동 포워딩 규칙 적용

**2) Gmail 라벨(폴더) 구성**
수집 Gmail에서 다음 라벨을 생성합니다: `AI/Action`, `AI/Partnership`, `AI/Updates`, `AI/Archive`, `AI/Inbox`
> **핵심**: "필터가 못 잡은 메일"이 AI/Inbox에 남게 만들면 보강 포인트가 보입니다.

**3) 광고/저우선 메일 필터 정리**
- **검색어**: `(광고 OR "AD" OR sponsored OR promotion OR promo) -subject:(보안 OR security OR 인증 OR otp OR 결제 OR payment OR 영수증 OR invoice OR 일정 OR 미팅 OR meeting OR 마감 OR due) -from:(accounts.google.com OR no-reply@accounts.google.com)`
- **동작**: 삭제(휴지통 이동) 및 일치하는 대화에도 필터 적용

**4) OpenClaw 메일 엔진 연결 (Himalaya)**
- 서버에서 Himalaya 설치 후 계정 연결: `~/.config/himalaya/config.toml` 생성 (Gmail 앱 비밀번호 연결)
- 확인: `/root/.local/bin/himalaya envelope list --page-size 10`

**5) OpenClaw 자동화 스크립트 구성**
- `scripts/mail_digest_labels.py`: Gmail 라벨 기준 분류 리포트 생성
- `scripts/mail_action_board.py`: 일정/할일/협업 후보 추출
- `scripts/mail_to_obsidian.py`: 옵시디언 일일 노트에 누적 저장
- `scripts/mail_todo_from_board.py`: 메일 Todo 인박스 노트 생성
- `scripts/mail_to_todoist.py`: Todoist 프로젝트로 자동 전송
- `scripts/mail_obsidian_autopush.sh`: 전체 실행 + Git 자동 커밋/푸시

**6) 옵시디언 저장 위치**
- **Vault**: `reedo-dev-vault`
- **권장 경로**: `20_Projects/메일분류/`
- **생성 문서 예**: `메일-점검-YYYY-MM-DD.md`, `메일-Todo-Inbox.md`, `메일-운영-지침-v1.md`

**7) Todoist 연동 (휴대폰 확인용)**
- Todoist 앱 설치 및 '메일 액션' 프로젝트 생성
- Developer 섹션에서 API token 복사
- 서버에 토큰 저장 (`~/.secrets/todoist_token`)

**8) 자동 실행(크론) 설정**
- 하루 2회 실행 예시 (09:00, 17:00 KST):
```bash
CRON_TZ=Asia/Seoul
0 9,17 * * * /root/.openclaw/workspace/scripts/mail_obsidian_autopush.sh
```

**9) 2단계 보강 (핵심 운영 최적화)**
- 휴대폰 Todo 과부하 방지: 모바일 전송은 P1/P2 고우선순위 한정
- P3 액션과 주요 정보는 옵시디언에만 저장 (분리 보관)
- 제목+발신자 중복 제거 로직 강화
- 제목 키워드로 기한(due) 자동 추정

### 💡 운영 팁

- `AI/Inbox`가 늘어나면 필터 보강을 우선적으로 진행하세요.
- 오탐이 많으면 제외 키워드(발신자/제목)를 강화합니다.
- 누락이 많으면 Action 키워드 조건을 완화합니다.
- 주 1회 5분 점검으로 지속적으로 정확도를 개선하세요.
